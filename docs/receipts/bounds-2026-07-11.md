# L1 canonical-gate bounds receipt — 2026-07-11

Records the measurement pass behind the release-gate performance floors/ceilings
introduced with the L1-bounds unit and enforced in `Sources/AgentSchema/AgentPackageSpec.swift`
(`AgentPackagePerformanceBudget`) plus the shell gate in `tools/verify-release.sh`.

## Framing: these are QUIESCENT canonical-gate floors, not load-invariant promises

Every number below was measured on an idle machine (system load average < 3.5),
holding the bench lock (`/private/tmp/.../bench.lock`) so no sibling job contended
for GPU/memory bandwidth. The floors are the bar the **release gate** must clear on
a quiescent box; they are **not** a guarantee of throughput under arbitrary load.

Concretely: a sibling unit later measured a single 0.8B decode rep at ~272 tok/s
while the machine was at load ~6 — below the 280 tok/s floor. That is **expected**
for a quiescent floor and is **not** a regression: the gate is defined at load < 3.5.
Do not "fix" the constants to accommodate contended measurements; re-measure quiescent.

## Method

- Margin envelope: **floors = worst-observed x 0.85** (conservative, ~15% headroom);
  **ceilings = worst-observed x 1.15**. Rounded conservatively (floors rounded DOWN,
  ceilings rounded UP) to round numbers.
- Text throughput: `bash tools/benchmark.sh <pkg> --use-package-profile --defer-startup-profile`
  (per-run decode / prefill64 / prefill256 tok/s from the package profile path).
- TTS latency/rtf: `bash tools/benchmark-voice-startup.sh <pkg> --use-package-profile --skip-build --warmup 0`
  (first_audio_s, ttfa_s, and rtf = generation-wall / produced-audio).
- A first text batch was **discarded**: taken at load 7-8, it read prefill256 ~30% low
  (bandwidth-starved). Re-measured at load < 3.5 under the bench lock; those quiescent
  numbers are the ones recorded here.

## Text SKUs

### Qwen/Qwen3.5-0.8B (10 runs, quiescent)

| metric        | worst observed (tok/s) | x 0.85  | floor (tok/s) |
|---------------|------------------------|---------|---------------|
| decode        | 332.1                  | 282.3   | **280**       |
| prefill64     | 2685.0                 | 2282.3  | **2280**      |
| prefill256    | 3061.5                 | 2602.3  | **2600**      |

### Qwen/Qwen3.5-2B (10 runs, quiescent)

| metric        | worst observed (tok/s) | x 0.85  | floor (tok/s) |
|---------------|------------------------|---------|---------------|
| decode        | 205.2                  | 174.4   | **170**       |
| prefill64     | 965.3                  | 820.5   | **820**       |
| prefill256    | 1010.5                 | 858.9   | **850**       |

Non-shipped models get no floor (only these two SKUs run the release gate).

## TTS (qwen3-tts, 8 runs, quiescent)

| metric        | worst observed | x 1.15 | ceiling      |
|---------------|----------------|--------|--------------|
| first_audio_s | 0.420 s        | 0.483  | **0.50 s**   |
| ttfa_s        | 0.407 s        | 0.468  | **0.50 s**   |
| rtf           | 0.984 x        | 1.132  | **1.20 x**   |

`rtf` (real-time factor = generation wall / produced-audio duration; < 1 => faster
than real time) is a new canonical required metric with unit `x`, enforced alongside
first_audio_s / ttfa_s at the `qwen3-tts.ttfa` gate.
